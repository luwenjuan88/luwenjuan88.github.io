Solidity编程（16）：函数可见性

发表于 2022-07-27 更新于 2022-07-29 分类于 Solidity Changyan：

对于方法的可见性的修饰，有四种修饰符：

public：方法在任何地方都可以调用

internal：类似于protect，在本合约能调用，子合约也能调用

private：只能在本合约调用

external：只能在外部调用，自己也不能调用

Solidity编程（17）：智能合约有哪些地址？

发表于 2022-07-27 更新于 2022-07-29 分类于 Solidity Changyan：

由40个字符的十六进制数字组成

在我们智能合约编程中，涉及到三种地址：

智能合约地址：智能合约部署到的地址

智能合约创建者的地址：创建智能合约的人的地址，一旦合约创建，就固定了

地址类型都有一个字段，叫做balance，可以直接访问查询余额。

Solidity编程（24）：案例｜向合约发送代币

发表于 2022-07-27 更新于 2022-08-29 分类于 Solidity Changyan：

当我们要使用智能合约操作从一个合约向另一个合约发送代币时

需要在代币合约上调用授权，也就是approve函数，把代币授权给我们才能有权操作。

比如

我们发行了一个代币MyToken

发行代币的时候往钱包A里空投了很多代币

然后我们新建一个合约MyContract

我们想要在合约MyContract里，将钱包A的代币发送到本合约

如果直接调用transfer发送，肯定是不行的，要不就可以随便偷别人钱包里的钱了。

如果想要发送代币，需要调用代币合约MyToken的approve函数，授权MyContract的地址可以操作的总代币数量，比如100。

授权完成后，我们在MyContract里就可以调用代币的transfer来发送代币到我们的合约了，不过总数量不超过100，多次发送相加不超过100。

其实就是代币合约MyToken授权我们的合约MyContract，作为某个代理，代为发送代币。

Solidity编程（26）：类库library

发表于 2022-07-27 更新于 2022-07-30 分类于 Solidity Changyan：

（1）不能被继承

（2）不能包含区块链的数据，也就是没有状态变量，可以有constant常量

（3）不能接受代币

（4）不能有构造函数

（5）部署一次，多次使用

（6）用得好的话，可以省gas

类似java的单类模式
